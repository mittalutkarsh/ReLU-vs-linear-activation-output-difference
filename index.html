S1 · Why nonlinearity matters

This integrated demo combines S1-1 and S1-2 on the same noisy two-ring dataset. You will see two distinct ideas: first, a single ReLU hidden layer bends the decision boundary around the ring; second, stacking more linear layers without activations changes nothing because the whole stack still collapses to one affine map.

S1-1

Activations exist for a reason.

A model with no hidden nonlinearity can only learn a straight decision boundary, so it cannot separate two concentric rings. Add one ReLU hidden layer and, with the same data and training recipe, the boundary bends around the inner ring.

No nonlinearity, no curve: a purely linear model can only draw a straight line, and one ReLU layer is enough to bend it around the ring.

S1-2

Depth without nonlinearity is a lie.

Five stacked linear layers still collapse to a single affine map, so they are no more expressive than one linear layer. On the ring task, both fail in the same way. Insert ReLUs between the exact same five layers, and the deep network suddenly becomes useful.

Stacked linear layers multiply into a single matrix, so five of them draw the same one line — until a ReLU between each layer unlocks the depth.

Model C · 1 linear layer

Linear(2 → 1) → Sigmoid. Baseline single affine boundary.

Accuracy —

Model D · 5 linear layers

2 → h → h → h → h → 1 with no activations. Still just one affine map.

Model E · 5 layers + ReLU

2 → h → h → h → h → 1 with ReLU between hidden layers. Same depth, now actually nonlinear.

Accuracy —

ReLU breaks the tie: now depth can carve a ring-shaped region.

Part C · Visual answer: same two numbers, different geometry
The resolution

You are right that both networks end with one logit, one probability — never “more than two numbers.” So the difference cannot be how many numbers come out. It is where the two numbers come from. Write the whole network at a single input as ŷ = sigmoid(w_x·x + w_y·y + b). That [w_x, w_y] is the local decision direction. Below, at each sampled location I compute that pair and draw a short line perpendicular to it — the local decision line the classifier draws right there.

Deep linear stack — one global line

Because W_eff = W_L × … × W1 is a fixed matrix, [w_x, w_y] is constant. Every little line points the same way.

Run the demo to sample the local linear maps.

ReLU stack — many local lines

Same 2D output at the end. But W_eff(x) = W_L·D_{L-1}(x)·…·D_1(x)·W1 has input-dependent gates, so [w_x, w_y] changes region to region and the lines rotate to wrap the ring.

The final scalar count is a red herring — what matters is whether those two numbers are allowed to vary across the input space.
S1-3

Embeddings learn similarity from nothing but next-token.

This model is never told that cat and dog are related. It is trained on one objective only: predict the next token in a tiny synthetic grammar. Because same-category words appear in the same slots, they end up with the same next-token distributions — and the embedding table quietly organizes them into clusters. The token IDs are deliberately shuffled, so ID adjacency cannot be the explanation. The only signal is co-occurrence.

Trained only to predict the next word, the model has to place same-category tokens close together — similarity falls out of the objective, never fed in.

How the grammar forces this (and why it is not cheating)

The training signal is purely next-token cross-entropy. Similarity emerges as a side effect of the optimization, because the softmax can only assign the same next-token distribution to two tokens if their embeddings point the same way.

Step 1 · The synthetic grammar
Categories in slots

Words are grouped into categories, then dropped into fixed templates. Same-category words are interchangeable within a slot, so they share the same right-context.

Retrain to show the grammar and templates.
Step 2 · The model
Embedding → softmax

Look up embeddings for the context, average them (CBOW) or take the single previous token (bigram), then a linear layer plus softmax predicts the next token. There is no similarity loss anywhere.

Retrain to show the model shapes.
Step 3 · Why similarity emerges
The mechanism

To predict the shared next-token distribution of a category, gradient descent has only one lever: make same-category embeddings similar. Distinct categories get distinct right-contexts, so they are pushed apart. The clustering is the optimum, not an accident.

S1-4

Memorization vs generalization, and data closes the gap.

A high-capacity network on tiny data will drive train loss to almost zero — it simply memorizes the points, label noise and all — while its held-out loss stays high. That distance between fitting the training set and actually generalizing is the generalization gap. The single most reliable way to close it is not a cleverer model but more data. Below, the exact same network is trained at three dataset sizes; capacity and optimizer budget are held fixed, so anything that changes is due to data alone.

Tiny data lets the network memorize (train loss → 0, test loss high); more data leaves nothing to memorize, so the gap collapses — same model, only the data changed.

The takeaway
